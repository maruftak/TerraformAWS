AWS RESOURCE TYPES QUICK REFERENCE
===================================

COMPUTE RESOURCES
-----------------

aws_instance
  - EC2 virtual machine
  - Required: ami, instance_type
  - Common: vpc_security_group_ids, subnet_id, key_name, user_data
  - Example: Web servers, app servers, databases

aws_launch_template
  - Template for EC2 instances
  - Used with Auto Scaling
  - Required: name or name_prefix
  - Common: image_id, instance_type, network_interfaces

aws_autoscaling_group
  - Manages EC2 instance scaling
  - Required: max_size, min_size
  - Common: desired_capacity, health_check_type, vpc_zone_identifier

aws_lambda_function
  - Serverless function
  - Required: function_name, role, handler, runtime
  - Common: environment, timeout, memory_size

aws_ecs_cluster
  - Container orchestration cluster
  - Required: name
  - Common: capacity_providers, settings

aws_ecs_task_definition
  - ECS task definition
  - Required: family, container_definitions
  - Common: cpu, memory, network_mode, requires_compatibilities

aws_ecs_service
  - ECS service
  - Required: name, cluster
  - Common: desired_count, launch_type, load_balancer

STORAGE RESOURCES
-----------------

aws_s3_bucket
  - Object storage bucket
  - Required: bucket
  - Common: tags, force_destroy

aws_s3_bucket_versioning
  - S3 bucket versioning
  - Required: bucket, versioning_configuration

aws_s3_bucket_server_side_encryption_configuration
  - S3 encryption
  - Required: bucket, rule

aws_ebs_volume
  - EC2 block storage
  - Required: availability_zone
  - Common: size, type, encrypted, snapshot_id

aws_ebs_snapshot
  - EBS snapshot
  - Required: volume_id
  - Common: description, tags

aws_efs_file_system
  - Elastic File System
  - Common: encrypted, performance_mode, throughput_mode

aws_efs_mount_target
  - EFS mount point
  - Required: file_system_id, subnet_id
  - Common: security_groups

NETWORKING RESOURCES
--------------------

aws_vpc
  - Virtual Private Cloud
  - Required: cidr_block
  - Common: enable_dns_support, enable_dns_hostnames

aws_subnet
  - VPC subnet
  - Required: vpc_id, cidr_block
  - Common: availability_zone, map_public_ip_on_launch

aws_internet_gateway
  - Internet gateway for VPC
  - Required: vpc_id
  - Common: tags

aws_nat_gateway
  - NAT gateway
  - Required: allocation_id, subnet_id
  - Common: tags

aws_route_table
  - Route table
  - Required: vpc_id
  - Common: route blocks

aws_route_table_association
  - Associate subnet with route table
  - Required: subnet_id, route_table_id

aws_security_group
  - Virtual firewall
  - Required: name or name_prefix
  - Common: description, vpc_id, ingress, egress

aws_security_group_rule
  - Security group rule
  - Required: type, from_port, to_port, protocol, security_group_id
  - Common: cidr_blocks, source_security_group_id

aws_lb (Application/Network Load Balancer)
  - Load balancer
  - Required: name, load_balancer_type
  - Common: subnets, security_groups, internal

aws_lb_target_group
  - Load balancer target group
  - Required: name, port, protocol, vpc_id
  - Common: health_check

aws_lb_listener
  - Load balancer listener
  - Required: load_balancer_arn, port, protocol, default_action
  - Common: ssl_policy, certificate_arn

aws_eip
  - Elastic IP address
  - Common: vpc, instance, network_interface

aws_route53_zone
  - DNS hosted zone
  - Required: name
  - Common: vpc, tags

aws_route53_record
  - DNS record
  - Required: zone_id, name, type
  - Common: ttl, records, alias

DATABASE RESOURCES
------------------

aws_db_instance
  - RDS database instance
  - Required: instance_class, engine
  - Common: allocated_storage, username, password, multi_az

aws_db_subnet_group
  - RDS subnet group
  - Required: name, subnet_ids
  - Common: description, tags

aws_db_parameter_group
  - RDS parameter group
  - Required: name, family
  - Common: parameter blocks

aws_rds_cluster
  - Aurora cluster
  - Required: cluster_identifier, engine
  - Common: master_username, master_password, database_name

aws_rds_cluster_instance
  - Aurora cluster instance
  - Required: identifier, cluster_identifier, instance_class, engine
  - Common: publicly_accessible

aws_dynamodb_table
  - DynamoDB NoSQL table
  - Required: name, hash_key
  - Common: billing_mode, range_key, attribute, global_secondary_index

aws_elasticache_cluster
  - ElastiCache cluster
  - Required: cluster_id, engine
  - Common: node_type, num_cache_nodes, parameter_group_name

SECURITY & IAM RESOURCES
------------------------

aws_iam_user
  - IAM user
  - Required: name
  - Common: path, tags

aws_iam_group
  - IAM group
  - Required: name
  - Common: path

aws_iam_role
  - IAM role
  - Required: name, assume_role_policy
  - Common: description, max_session_duration

aws_iam_policy
  - IAM policy
  - Required: name, policy
  - Common: description, path

aws_iam_role_policy_attachment
  - Attach policy to role
  - Required: role, policy_arn

aws_iam_user_policy_attachment
  - Attach policy to user
  - Required: user, policy_arn

aws_iam_instance_profile
  - EC2 instance profile
  - Required: name, role
  - Common: path

aws_kms_key
  - KMS encryption key
  - Common: description, deletion_window_in_days, enable_key_rotation

aws_kms_alias
  - KMS key alias
  - Required: name, target_key_id

aws_secretsmanager_secret
  - Secrets Manager secret
  - Required: name
  - Common: description, kms_key_id

aws_secretsmanager_secret_version
  - Secret version
  - Required: secret_id, secret_string or secret_binary
  - Common: version_stages
